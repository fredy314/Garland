# 🎄 ESP32 Smart Garland (Розумна гірлянда)

Цей проєкт — розумна новорічна гірлянда на базі мікроконтролера ESP32, написана з використанням фреймворку ESP-IDF C++.

## ✨ Особливості та функціонал

- **Керування через Веб-інтерфейс:** Зручна панель управління у браузері телефону або комп'ютера.
- **Ефекти та режими:**
  - Постійне світіння
  - Почергове плавне
  - Дихання
  - Хаос
  - Свічка
- **Контроль яскравості та швидкості:** Гнучке налаштування поточного ефекту.
- **Збереження стану (NVS):** Обраний режим, яскравість та швидкість зберігаються у пам'ять ESP32 і автоматично відновлюються після перезавантаження.
- **Нічний режим:** Можливість налаштувати гірлянду так, що вона буде автоматично вмикатися лише після заходу сонця (час синхронізується по SNTP) та вимикатися з його сходом.
- **Інтеграція з Home Assistant (MQTT):**
  - Автоматичне виявлення в мережі (MQTT Discovery) завдяки `johboh/homeassistantentities`.
  - Керування увімкненням, яскравістю та режимами роботи безпосередньо як пристроєм типу `Light` (з ефектами).
  - Окремий перемикач `Switch` для активації логіки нічного режиму («Тільки темрява»).
  - Повзунок швидкості анімації вбудований як об'єкт `Number`.
  - Унікальний `deviceId` на основі MAC-адреси.
- **Менеджер Wi-Fi:** 
  - Пріоритетне підключення до заданої Wi-Fi мережі (вибирає роутер з найсильнішим сигналом, якщо їх декілька).
  - Створення власної точки доступу (SoftAP `Garland_XXXXXX`), якщо домашня мережа недоступна.
  - Налаштовані паузи (2с після старту, 10с між реконектами) для стабільної роботи інших радіо-сервісів.
- **ESP-NOW Синхронізація часу:**
  - Гірлянди без інтернету можуть отримувати час від сусідніх гірлянд, які вже мають точний час (через NTP або від інших вузлів).
  - **Агресивне сканування:** вузол без часу послідовно проходить через всі 13 каналів, роблячи широкомовні запити.
  - **Автономність:** дозволяє логіці «Тільки темрява» (нічному режиму) працювати навіть у віддалених точках без доступу до Wi-Fi роутера.
- **Файлова система SPIFFS:** HTML, CSS та JavaScript файли зберігаються окремо від прошивки у flash-пам'яті мікроконтролера.
- **OTA Оновлення (Бездротове оновлення):**
  - Можливість оновлювати як саму прошивку (`.bin`), так і файли веб-інтерфейсу (`storage.bin`) безпосередньо через браузер.
  - **Безпека:** Сторінка оновлення доступна за адресою `/ota.html` та захищена паролем. Налаштувати логін та пароль можна у файлі `main/WebServerManager.h` (константи `OTA_USER` та `OTA_PASS`). За замовчуванням: `admin` / `31415926`.
  - **Верифікація прошивки:** Система перевіряє, чи призначений файл саме для цього пристрою (Project ID: `Garland`).
  - **Захист від відкату (Downgrade Protection):** Забороняє випадкове завантаження старішої версії прошивки (можна вимкнути за допомогою спеціального чекбокса).

## 🛠 Збірка та встановлення

### Вимоги
- Налаштоване середовище **ESP-IDF** (v6.x рекомендується).
- Плата ESP32 (змінні піни можна налаштувати у `main/main.cpp`).

### 1. Підготовка
Проєкт налаштовано на використання кастомної таблиці партицій `partitions.csv` та 4MB флеш-пам'яті (через `sdkconfig.defaults`). 

Версія прошивки задається у головному файлі `CMakeLists.txt`:
```cmake
set(PROJECT_VER "1.0.0")
```

### 2. Налаштування Wi-Fi та MQTT
У файлі `main/main.cpp` знайдіть `SSID` та `PASSWORD` і змініть їх на власні дані від домашньої мережі для автоматичного підключення під час запуску.  
Також за потреби змініть адресу MQTT-брокера (`mqtt.lan`) у файлі `main/MqttManager.cpp`.

### 3. Компіляція та Прошивка
Для зручності використовується скрипт `compile.sh` (в Docker контейнері):

```bash
# Збірка та прошивка
./compile.sh flash
```

## 📱 Використання
1. Після включення гірлянда спробує підключитися до вказаного Wi-Fi роутера. 
2. IP-адресу гірлянди можна знайти в моніторі порту або Home Assistant.
3. Якщо роутер недоступний, підключіться до Wi-Fi мережі `Garland_XXXXXX` та відкрийте `http://192.168.4.1`.
4. Для оновлення прошивки перейдіть на сторінку `http://<IP-адреса>/ota.html`.
